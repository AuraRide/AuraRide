## 1. 产品一句话

**AuraRide —— 把心情骑成一条路线。**
用户写下此刻的心情 → 系统解析成一种"情绪颜色" → 据此在周边推荐三条真实骑行路线 → 边骑边拍下沿途瞬间 → 沉淀为一份按颜色归档的"情绪地图"与可回看的旅程动画。

**差异点:** 市面骑行 App(Strava/Keep/高德骑行)都在比"数据/竞技";我们卖的是**情绪 × 城市骑行 × 影像记忆**的情感闭环。这是评委和用户记得住的钩子。

---

## 2. 参赛定位与成功标准

- **赛题要求:** 开发一款可上架 iOS App Store 的 App → 必须是**真机可装、可走完整流程、符合 Apple 审核规范**的原生(或原生壳)应用,不能只是网页 demo。
- **拿奖的关键不是功能多,而是:**
  1. 一个**完整顺滑、零崩溃**的核心闭环(心情→路线→骑行→记录→回顾)
  2. 一个**记得住的差异化故事**(情绪地图)
  3. **技术可信度**(真实 GPS / 地图 / 后台记录 / 原生能力)
  4. **设计完成度**(视觉、动效、无障碍、空错载态)
  5. 一场**讲得清的 demo**
- **可量化目标(自评打分用):** 核心流程成功率 100%、冷启动 < 2s、骑行页 ≥ 50fps、崩溃率 0、关键页均有空/错/载入态、通过 Apple 审核 checklist 自检。

## 3. 目标用户与场景

- **主用户:** 18–35 岁城市青年,骑行通勤/休闲,重视情绪表达与生活记录(小红书式人群)。
- **核心场景:** 心情不好/想放空 → 打开 App 写一句 → 跟着推荐路线骑一段 → 拍几张 → 回家看回顾。
- **非目标(v1 不做):** 专业竞速、社交社区、多人约骑。

## 4. 现状盘点(已完成)

技术栈:React + Vite + react-router + framer-motion + Tailwind + 高德 JS API,数据存 localStorage。

| 模块 | 状态 |
|---|---|
| 进入流程:登录/首屏 → 写心情 → 颜色揭晓 | ✅ 网页版完成(登录是**前端假登录**) |
| 心情→颜色引擎(本地关键词/情感) | ✅ `lib/moodColor.ts`(可后续换 LLM/情感 API) |
| 周边三条真实路线推荐 | ✅ 高德 AMap.Riding 真实路网 + GCJ-02 转换 |
| 骑行 HUD:真实 GPS + 高德地图 + 轨迹 | ✅ 网页版(后台/锁屏会暂停;心率是模拟) |
| 拍照记录 / 骑行回顾 | ✅ 网页版(照片以 base64 存 localStorage) |
| 我的旅程:列表 / 按颜色 / 日周月年总结 + 情绪光谱 | ✅ |
| 旅程全览(照片动画故事页) | ✅ |
| 桌面手机框预览、演示数据(dev) | ✅ |

**底层现实:** 目前是**网页应用**,不是可上架的 iOS App;无真实后端/鉴权;数据只在浏览器本地;后台定位、相机、心率等**原生能力未接**。

## 5. 差距分析:距离"正式可上架 App"还差什么

### A. 工程化 / 原生壳(最硬的骨头)
- [ ] 选定打包方案(见决策⚠️1)并搭起 iOS 工程:Xcode 项目、Bundle ID、签名证书、App Store Connect、TestFlight 内测。
- [ ] 原生能力桥接:**后台定位**(锁屏继续记录,骑行 App 的命门)、**相机**、Haptics、推送、(可选)HealthKit 心率。
- [ ] 启动屏(LaunchScreen)、App 图标(全尺寸)、深浅色适配。

### B. 数据 / 持久化 / 后端
- [ ] **照片不能存 localStorage**(base64 很快撑爆配额)→ 迁移到原生文件系统 / SQLite。
- [ ] 骑行记录持久化方案:本地优先(推荐)还是云同步(见决策⚠️3)。
- [ ] 鉴权:目前是假登录。若提供微信等第三方登录,**Apple 强制要求同时提供 Sign in with Apple**(见决策⚠️4)。

### C. 权限 / 隐私 / 合规(审核硬门槛)
- [ ] Info.plist 权限文案:定位(使用期间/始终)、相机、相册、运动与健身、通知 —— 缺了会被拒。
- [ ] **权限预说明**(先讲为什么再弹系统框),提升授权率、符合 Apple 指南。
- [ ] **隐私政策 + 用户协议**(需有可访问 URL)、App Store「隐私营养标签」如实填写。
- [ ] 情绪/位置属敏感数据 → 数据最小化、可删除、可导出(也是卖点)。
- [ ] 高德**商用授权**与配额;字体(serif-cn)**商用版权**确认。

### D. 核心功能完整度 / 健壮性
- [ ] 骑行:后台/锁屏持续记录、暂停/继续、自动暂停、距离/时长/海拔/均速口径校准。
- [ ] 相机:真实拍照 + 自动打上位置/颜色标签 + 写入相册(可选)。
- [ ] 全量**空态 / 错误态 / 加载态 / 无网络态**;定位被拒、无路线、无照片的兜底。
- [ ] 心率:接 HealthKit 或蓝牙;否则 v1 明确"暂不支持/选填"。
- [ ] 清理遗留路由与死代码(/deep-sea、/healing、/burnout、/trace、/phone、旧 /ride 等)。

### E. 设计 / 无障碍 / 体验
- [ ] **统一设计系统**:色板/字阶/间距/圆角/动效规范 + Figma 源文件(你主导)。
- [ ] **首页重构**(你正想做的)纳入设计系统统一推进。
- [ ] 无障碍:动态字体(Dynamic Type)、VoiceOver、对比度、**降低动态效果**(很多页面无限动画,需尊重系统设置)。
- [ ] Haptics(触感反馈已部分用,系统化)、必要的声音设计。

### F. 上架素材 / 提交
- [ ] App 名称、副标题、关键词、描述、分类、年龄分级。
- [ ] 各机型截图、(加分)预览视频、1024 图标。
- [ ] 审核备注 + 测试账号(若有登录)、隐私政策/支持 URL、合规自检。

### G. 质量保障
- [ ] 崩溃监控(如 Sentry)、轻量埋点(看核心漏斗)、关键路径手测清单、性能预算。

## 8. 竞争力:从哪几方面拉开差距

1. **差异化叙事(最重要):** 把"情绪→颜色→路线→影像→回顾"的闭环讲成一个完整故事。所有文案、视觉、demo 都围绕"情绪地图 / 把心情骑出来"。评委记住的是故事,不是功能列表。
2. **传播性:** 单次骑行卡片、年度/照片故事一键导出图片或短视频 —— 用户自发分享 = 免费曝光,也是 demo 高光。
3. **留存机制:** 每日心情打卡 + streak + 周期总结 + 通知,形成"情绪日记"习惯。
4. **技术可信度(评委加分项):** 后台实时 GPS、灵动岛/Live Activity、真实高德路线规划(已做)、离线可用、HealthKit —— 任选 1–2 个做扎实。
5. **体验完成度:** 无障碍 + 空错载态 + 60fps + Haptics,体现"正式产品"而非 demo。
6. **隐私可信:** 本地优先 + 透明权限 + 可导出可删除,正好契合"情绪"这种敏感数据,是差异化卖点。

> 策略:**P0 保闭环零瑕疵**,再从 #1/#2/#4 里各挑一个做成"高光时刻",而不是平均用力。
