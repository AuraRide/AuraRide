# AuraRide 产品骨架(信息架构 · 页面 & 功能全集)

> 驱动原则:**色彩是这款产品唯一的主线与最大卖点。** 每个页面都要回答一个问题——"此刻/这段时间,你是什么颜色?"
> 标记:🟢已有 · 🟡增强 · 🔵新增 ·(☁️需后端)·【阶段】P0=初赛/演示就绪 / P1=显著加分 / P2=赛后或决赛

---

## 0. 色彩主线(贯穿全局的设计准则)

1. **一次骑行 = 一种颜色**(由出发心情解析而来)。
2. **一段时间 = 一种颜色**(把多次骑行的颜色聚合成"周颜色 / 月颜色 / 年度颜色")。←本次新增的升维
3. 颜色不仅是装饰,而是**导航与回顾的语言**:可按颜色浏览照片、按颜色回看路线、用颜色总结人生的一段路。
4. 所有总结/分享物料一律**以颜色为主视觉**,可分享、可传播。

---

## 1. 顶层导航结构

- **仪式主线(沉浸、线性,无常驻导航):**
  首页 → 写心情 → 颜色揭晓 → 路线推荐 → 骑行中 → 骑行回顾(含骑后复盘)→ 回到首页
- **我的旅程 Hub(从首页左上"旅程"药丸进入,分区/Tab):**
  每次骑行 · 按颜色 · 周期总结 · **我的色彩(新)** · 旅程全览
- **色彩档案分支:** 我的色彩 → 周颜色 / 月颜色 / **年度回顾(全屏故事)**
- **影像与分享:** ins 风格相册 → 分享卡片(单次 / 周月年色卡 / 相册拼图)
- **社区(后端阶段):** 发现/广场 ← 分享路径
- **系统:** 登录/账号 · 设置 · 首启引导

## 3. 跨页面功能清单
- **色彩聚合引擎**(周/月/年代表色 + 色谱)——见 §5,本次新增的核心逻辑。
- **分享导出**(html→图片,如 html2canvas;视频可选)。
- **骑后心情复盘**(记录前/后心情,产出"情绪变化"独有数据)。
- **数据 / 存储**:本地优先;照片存文件系统(非 base64);骑行轨迹持久化。
- **地图 / 定位**:高德真实路网 + GCJ-02;后台记录(决赛)。
- **相机**:真实拍照 + 颜色/位置标签。
- **通知 / 打卡 / streak**。
- **无障碍**:动态字体 / VoiceOver / 降低动态效果。

## 4. 数据模型(骨架)
- **Ride** { id, startedAt, endedAt, distance, duration, path(轨迹点[]), colorId, dominantColor(hex), moodTextBefore, moodAfter?, photos[] }
- **Photo** { id, fileUri, color(hex), caption?, location?, takenAt }
- **ColorPeriod**(派生,不存) { scope: week|month|year, repColor(hex), palette:[{colorId,hex,weight}], rides, distance, photos, topEmotion }
- **MoodCheckin** { beforeText, beforeColorId, afterColorId?, afterScore?, note? }
- **SharedPath**(☁️云) { id, author, rideSummary, path, repColor, photos[], caption, likes }

---

## 5. 色彩聚合定义(卖点的"算法",要可信也要好看)
- **代表色:** 取该时段所有骑行的情绪色,按**里程(或次数)加权混合**(建议在 HSL/Lab 空间混合,得到悦目的单一代表色);同时**保留完整色谱**(各情绪色占比,复用现有"情绪光谱")。
- **周颜色 / 月颜色:** 代表色 + 色谱 +(月)逐周漂移条。
- **年度颜色:** 年代表色 + **12 个月色带** + 年度主色情绪 + 全年轨迹 + 精选相册。
- 文案模板:「本周你偏向__」「__月,你大多是__色」「2026,你的主色是__」。

## 6. 给程序同学的"先搭什么"(并行安全 · 不依赖最终设计稿)
> 全部用现有原型当行为规范、现有样式当占位皮肤;视觉最后"换皮"(token 化)。

1. **数据模型扩展 + 存储迁移**(moodAfter、轨迹持久化、照片→文件系统)。
2. **色彩聚合工具**(§5,纯函数,可写单测)——这是新卖点的地基。
3. **骨架页面(结构先行,样式占位):** 我的色彩 D1 / 周颜色 D2 / 月颜色 D3 / 年度回顾 E1(复用 Overview 的场景序列引擎)/ ins 相册 F2 / 分享卡片 F3。
4. **骑后心情复盘 B4** + **分享导出 F3**(html2canvas)。
5. **路线偏好/环线 B1**、清理遗留路由。
6. **社区 G1/G2 ☁️** 留后端阶段;现在先做**本地精选示例**撑起演示形态。
7. 全程**样式 token 化**,等设计稿到位换皮,逻辑不动。

**优先级建议(冲文档/截图/复赛视频):** 先做 §6 的 2→3(色彩聚合 + 色彩档案/年度回顾/相册/分享卡)——它们既是"色彩"卖点的集中体现,又是文档配图、海报、视频里最出彩的素材,且**全部本地可跑、无需后端**。社区(需后端)留到后面。
