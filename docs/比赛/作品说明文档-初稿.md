## 1. 问题背景与用户分析(目标 200 字)

城市青年正承受普遍的情绪压力,却缺少低门槛、能落到行动上的情绪疏导出口。骑行本是大众化的解压方式,但市面骑行类 App 几乎都聚焦速度、里程、排名与打卡,把骑行变成又一项"任务",反而加重焦虑;与此同时,年轻人渴望记录与表达情绪与生活,却找不到有温度的载体。

目标用户为 18–35 岁城市青年——通勤与休闲骑行者,熟悉智能手机与社交分享(典型如小红书人群)。其核心痛点有三:心情低落时想骑车放空却不知去哪;骑行过程缺乏有情感的记录方式;现有运动 App 冰冷功利,只见数据不见心情。AuraRide 面向"想用骑行安放情绪、并把它温柔记下来"的人群。

## 2. 相关竞品分析(目标 200 字)

主流运动/骑行 App(Strava、Keep、咕咚、高德骑行)以数据、路书、竞技与社交见长,工具属性强,但情绪价值薄弱、记录冰冷,本质是"更努力地运动";情绪/心理类 App(各类心情日记、冥想助眠应用)虽关注情绪,却与真实户外行动割裂,停留在"屏幕内的记录"。

两类产品之间存在明显空白:几乎没有人把"情绪—真实骑行—影像记忆"真正打通。AuraRide 正切入这一交叉地带——不与运动 App 拼数据,也不做纯粹的心情记录,而是用一次真实的骑行,为情绪提供出口与容器。这一差异化定位避开红海竞争,形成清晰的认知壁垒。

## 3. 可行性分析(目标 300 字)

**技术可行:** 产品已基于 React 与高德地图开放平台完成可运行的完整原型,跑通"进入→写心情→生成情绪颜色→周边三条真实路线→实时骑行→拍照记录→旅程回顾"全流程。其中骑行路线由高德骑行路径规划(AMap.Riding)基于真实路网生成,并完成 WGS-84→GCJ-02 坐标合规转换;定位与轨迹来自设备真实 GPS;"心情→颜色"采用本地轻量情感引擎,可平滑升级为情感 API 或大模型。数据本地存储,隐私友好、离线可用。

**工程可行:** 核心闭环已验证,后续可用 Capacitor 将现有前端平滑封装为 iOS App,复用全部界面、无需重写;后台定位、相机、HealthKit 等原生能力均有成熟方案。

**团队与成本可行:** 2 人(设计 + 开发)分工清晰;数据由地图开放平台与设备传感器自取,无外部数据依赖;地图免费额度叠加"本地优先"架构,开发与运营成本低,适合学生团队分阶段推进。

## 4. App 创新点(目标 300 字)

**概念创新——"情绪地图":** 首创将抽象心情解析为一种"情绪颜色",再据此在周边推荐真实骑行路线,构建"情绪→颜色→路线→影像→回顾"的情感闭环。骑行由"运动任务"转化为"情绪的出口与容器",这是本作与所有竞品的根本区别。

**交互与视觉创新:** 以"颜色"作为贯穿全程的情感线索;支持"一句话"即生成路线,门槛极低;流体色彩与沉浸式骑行界面强化情绪沉浸;旅程以"情绪光谱"图谱与"照片故事动画"(类年度报告)沉淀,把冰冷数据转译为可回味、可分享的情感影像。

**体验与技术创新:** 非竞技、非打卡,主打松弛感;对情绪这类敏感数据坚持隐私优先、本地存储;底层以真实路网骑行路线规划 + 真实 GPS 轨迹 + 国内坐标合规为支撑,保证落地可信。

## 5. 应用前景(目标 200 字)

AuraRide 契合大赛推荐的健康、智慧交通与社会创新领域,呼应城市青年心理健康与绿色低碳出行的双重趋势,用户需求真实且持续增长。其差异化定位避开运动工具红海,以情绪价值与影像记忆形成壁垒,并天然具备社交传播力——可分享的骑行卡片与年度照片故事能驱动自发扩散。

商业上具备多元想象:情绪主题皮肤与路线、文旅与单车品牌联名、城市骑行内容生态;产品亦可横向拓展至跑步、散步、通勤等泛户外情绪记录,并深度融入 Apple 生态(Apple Watch、灵动岛、HealthKit),具备从校园作品成长为长期产品的潜力。
